Upload Vulnerablities


Evading filters is often the challenging aspect of upload vulnerablitites. The following are the most common filters.
-Client Side
Bypass:
Intercept with burp (refresh with ctrl+F5 to not use cache)(be sure to not ignore .js files) and edit the js filter. Then turn off intercept.
Try upload something legit, catching the POST request with burp, then replace the legit thing with your code.
-Server Side
File Extensions:
Determine it if is a white or blacklist by uploading yeet.notarealextension. If that is blocked, you'll know it is a whitelist.
php extensions:
.php, .phtml, .php3, .php4, .php5, .php7, .phps, .php-s, .pht, .phar
Magic Byte:
Upload a legit thing, but change the magic byte (using hexeditor) to see if that is being filtered.
https://en.wikipedia.org/wiki/List_of_file_signatures
MIME Type:
Catch legit thing with burp and change the MIME type (text/x-php) to see if it is being filtered.
File Length
Try progressivley larger/smaller files to determine the limits.
