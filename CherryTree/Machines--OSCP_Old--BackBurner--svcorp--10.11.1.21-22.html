Nmap showed that many ports were open, most interestingly 21 (ftp), 80 (http), 135, 139, 445 (smb) and 3389 (rdp).
Visiting the web page on port 80 read the following, ‘All documents must be supplied for edit before final submission. The editorial team leads this process and will collect any document supplied. To submit a document connect to the FTP server on port 21 using the username: "editor" and the password "MyEditWork". Ensure that any documents are saved in the ".doc" or ".docx" format, this will allow our editors to quickly go through the document and perform editorial work’
The tester (Chris) confirmed that the FTP credentials were functional and that he was able to “put” to the server.
Chris used a msfvenom command to create a vbs script that he added to a macro on a word document named 443-167.docm.
He then uploaded the .docm file to the FTP server and recieved a reverse shell when the command was executed.
The shell he recieved was of alice/svcorp which allowed him to read the proof.txt file on the administrator desktop.
