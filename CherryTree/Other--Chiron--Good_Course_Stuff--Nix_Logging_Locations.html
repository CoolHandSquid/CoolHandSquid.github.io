Nix Logging Locations


Common Nix Logging locations:
-----------------------------

/var/log/apache2/access.log - ascii
- Web service access (User-ageent strings associated with clinet connections to web service)

/var/log/lastlog - data
- Tracks last successful login for accounts

/var/log/auth.log - ascii
- Login events

/var/log/secure - ascii
- Authorizations - system usage - 'sudo','ssh',etc

/var/log/wtmp - data
- curent/last login of users, kinda like status/history file for "utmp"

/var/run/umtp - data
- tracks users on system based on TTY/PTY

/var/log/btmp - data
- Failed login attempt info

"last" pulls last login attempts from wtmp
"lastb" pulls from btmp

Binary Log Editing:
---------------------
- w/ tty/pty enabled shell:
- Stop Service (if possible):
xxd /path/to/logfile | vi -

Depending on the log file, you may need additional tool on target - like 'wted' to edit 'wtmp' log file

Ascii Log Editing:
---------------------
sed -i '/add.res.sof.sys/d' /path/to/log/file
(removing string from file based on matched pattern - will work for sepcific times, user account names, etc. depending on which ascii log you're targeting)
